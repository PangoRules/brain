# Chapter 25 — Starting a Project (Variables, Constants, Here Docs)

This chapter starts building a real script — an HTML system report generator. The actual project is a vehicle for introducing **variables**, **constants**, and **here documents**.

---

## Variables

### Assignment

```bash
name=value          # no spaces around = (this is mandatory)
name="a string"     # quotes needed when value contains spaces
```

The shell creates a variable the moment you use it — no declaration needed. This means typos silently create empty variables instead of erroring. Use `set -u` (undefined variable error) to catch them.

```bash
foo="hello"
echo $foo           # hello
echo $fool          # (empty — shell silently made a new variable 'fool')
```

### Value types (everything is a string, but can hold these)

```bash
a=hello                     # plain string
b="hello world"             # string with spaces — requires quotes
c="value is $a"             # variable expansion inside double quotes
d="$(date +%Y-%m-%d)"       # command substitution
e=$(( 5 * 7 ))              # arithmetic expansion — result: 35
f="\t\tindented\n"          # escape sequences
```

### Naming rules

- Letters, numbers, underscores only
- First character must be a letter or underscore
- Case-sensitive: `foo` ≠ `FOO`
- **Convention:** `UPPERCASE` for constants/env vars, `lowercase` for local variables

### Brace syntax — avoid ambiguity

```bash
filename="report"
touch "${filename}.txt"       # → report.txt  (correct)
touch "$filename.txt"         # → report.txt  (works here, but...)
mv "$filename" "$filename1"   # WRONG: shell looks for var 'filename1' (empty)
mv "$filename" "${filename}1" # correct: appends literal 1
```

Always use `${var}` when the variable name is followed immediately by alphanumeric characters.

### Always quote variables

```bash
cp "$file" "$destination"   # safe even if paths have spaces
cp $file $destination       # breaks if either contains spaces
```

Double quotes allow expansion; single quotes suppress all expansion:

```bash
echo "$USER"    # → pango
echo '$USER'    # → $USER  (literal)
```

## Constants

Shell has no true constants by default — convention is `UPPERCASE` names. To enforce immutability:

```bash
declare -r TITLE="System Information Report"
# or in zsh:
readonly TITLE="System Information Report"

TITLE="changed"   # → error: TITLE: readonly variable
```

**Useful built-in variables for scripts:**

| Variable    | Value                                      |
| ----------- | ------------------------------------------ |
| `$HOSTNAME` | Machine's network hostname                 |
| `$USER`     | Current username                           |
| `$HOME`     | Home directory path                        |
| `$PWD`      | Current working directory                  |
| `$SHELL`    | Path to current shell (`/usr/bin/zsh`)     |
| `$PATH`     | Colon-separated executable search path     |
| `$RANDOM`   | Random integer 0–32767 (each access)       |
| `$LINENO`   | Current line number in the script          |
| `$0`        | Script name                                |
| `$?`        | Exit status of last command                |

## declare / typeset

```bash
declare -r VAR="value"    # read-only (constant)
declare -i VAR=42         # integer (arithmetic ops without $(( )))
declare -l VAR="HELLO"    # lowercase: stored as "hello"
declare -u VAR="hello"    # uppercase: stored as "HELLO"
declare -a VAR            # indexed array
declare -A VAR            # associative array (hash)
declare -p VAR            # print variable with its attributes
```

> In zsh, `declare` and `typeset` are interchangeable. Both work the same way.

## Here Documents

Embed a block of text directly in a script and feed it to a command's stdin. Cleaner than multiple `echo` lines.

```bash
command << DELIMITER
line 1
line 2
line 3
DELIMITER
```

- The delimiter (`_EOF_`, `END`, `HEREDOC`, anything) must appear **alone on its own line**
- **No trailing spaces** after the closing delimiter
- Variable expansion and command substitution happen inside (like double quotes)
- Single and double quotes inside lose their special meaning — no escaping needed

```bash
cat << _EOF_
<html>
  <head><title>$TITLE</title></head>
  <body>
    <h1>$TITLE</h1>
    <p>Generated by $USER on $(date)</p>
  </body>
</html>
_EOF_
```

### Suppress expansion (quote the delimiter)

```bash
cat << '_EOF_'        # quoted delimiter — NO expansion inside
echo $HOME            # printed literally as: echo $HOME
_EOF_
```

### Indented here doc (`<<-`)

`<<-` strips leading **tabs** (not spaces) from the body and closing delimiter — lets you indent for readability:

```bash
if true; then
    cat <<- _EOF_
	Line 1       ← this line is tab-indented
	Line 2
	_EOF_        ← closing delimiter can be indented too
fi
```

> Most editors use spaces, not tabs, which makes `<<-` annoying in practice. If your editor uses spaces, stick with `<<` and don't indent the heredoc body.

### Here doc to a file

```bash
cat << _EOF_ > output.html
<html>...</html>
_EOF_
```

### Here doc with other commands

```bash
# Pass commands to ssh
ssh user@host << _EOF_
cd /var/log
tail -20 syslog
_EOF_

# Pass commands to a database client
mysql -u root << _EOF_
USE mydb;
SELECT * FROM users;
_EOF_

# Supply input to a script expecting prompts
./interactive_script << _EOF_
yes
/tmp/output
_EOF_
```

## The Project So Far

```bash
#!/usr/bin/env bash
# sys_info_page — generates an HTML system report
# Usage: sys_info_page > report.html

set -euo pipefail

# Constants
TITLE="System Information Report For $HOSTNAME"
CURRENT_TIME="$(date +"%x %r %Z")"
TIMESTAMP="Generated $CURRENT_TIME, by $USER"

cat << _EOF_
<html>
  <head>
    <title>$TITLE</title>
  </head>
  <body>
    <h1>$TITLE</h1>
    <p>$TIMESTAMP</p>
  </body>
</html>
_EOF_
```

```bash
chmod +x ~/.local/bin/sys_info_page
sys_info_page > ~/report.html
xdg-open ~/report.html          # opens in Firefox on your setup
```

## Common Pitfalls

```bash
# WRONG: spaces around = break assignment
name = "hello"       # → tries to run command 'name' with args '=' 'hello'
name ="hello"        # → error
name= "hello"        # → sets name to "" then runs 'hello' as a command

# RIGHT
name="hello"

# WRONG: unquoted variable with spaces
cp $source $dest     # breaks if source contains spaces

# RIGHT
cp "$source" "$dest"

# WRONG: typo creates silent empty variable
echo $TITTLE         # (empty — should be $TITLE)

# FIX: add to top of script
set -u               # makes undefined variable references an error
```

## Quick Reference

```bash
var=value                   # assign
echo $var                   # expand
echo "${var}suffix"         # expand with brace (avoid ambiguity)
echo "$var"                 # always quote in practice
readonly var                # make read-only
declare -r var="val"        # same, with declare
unset var                   # delete variable

# Here doc
cat << EOF
text with $expansion
EOF

# Here doc, no expansion
cat << 'EOF'
literal $text
EOF
```
