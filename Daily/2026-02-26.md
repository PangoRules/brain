# 🌐 HTTP & API Testing Tools

## Recommendation

- **xh** → Best for interactive use  
- **hurl** → Best for scripted / CI testing  

---

## Tool Comparison

| Tool       | Install                         | Best For                                      |
|------------|--------------------------------|-----------------------------------------------|
| curl       | Pre-installed                  | Scripts, automation                           |
| xh         | cargo install xh               | Interactive use — HTTPie syntax, Rust speed   |
| httpie     | pip install httpie             | Similar to xh, Python-based (slower)          |
| hurl       | cargo install hurl             | Scripted test suites, assertions, CI          |
| bruno CLI  | npm i -g @usebruno/cli         | Team collections stored as plain Git files    |

---

# 🚀 xh Quick Examples

```bash
xh POST api.example.com/users name=John email=john@example.com
xh GET api.example.com/users Authorization:"Bearer mytoken"
```

---

# 🧪 hurl Example (Postman-like Test Suite in a File)

`test.hurl`

```hurl
POST https://api.example.com/users
Content-Type: application/json

{ "name": "John" }

HTTP 201
[Asserts]
jsonpath "$.id" isNumber
```

Run it:

```bash
hurl test.hurl
```

# 🧱 C# Unit Tests (.NET)

## Run from Terminal

```bash
dotnet test                                   # Run all tests
dotnet test --filter "MyTestClass"             # Filter by class
dotnet test --filter "MyMethod" -v detailed    # Specific test + verbose
dotnet watch test                              # Watch mode (re-run on save)
```

---

# 🧠 Neovim Integration

Add to `.nvim.lua`:

```lua
local term = require("custom.terminal")

vim.keymap.set("n", "<leader>tt", function()
  term.float_term("dotnet test --verbosity detailed")
end, { desc = "Run all tests" })

vim.keymap.set("n", "<leader>tw", function()
  term.float_term("dotnet watch test")
end, { desc = "Watch tests" })
```

# 🐞 Debugging a Specific Test with DAP

Your dotnet profile uses **netcoredbg**.

### Steps:

1. Set breakpoint in test method → `<leader>db`
2. Press `<F5>` to launch
3. Step through:
   - `<F10>` → Step over  
   - `<F11>` → Step into  
   - `<F12>` → Step out  
   - `<leader>du` → Open DAP UI

## 🔎 DLL Auto-Detection

DAP searches:

```
bin/Debug/net*/*.dll
```

Works for test projects as long as:

- The test project DLL name matches the project name.
